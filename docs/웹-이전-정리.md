# 아이로 웹앱 이전 정리 (2026-07-20)

React Native + Expo 앱(`iLaw-frontend` / 로컬 `airo-app`)을 **순수 웹앱**(`iLaw-web` / 로컬 `airo-web`)으로 이전.
백엔드(`iLaw-backend`)는 **변경 없이 그대로 공유**.

## 왜 웹으로?
- 앱스토어 출시를 접기로 함 → React Native가 더 이상 필요 없음
- `react-native-web`이 네이티브와 다르게 렌더돼서 "앱 다르고 웹 다르고" 이중 수정 발생 → 웹 단일 타겟으로 제거
- `eas deploy`는 배포마다 URL이 바뀜 → **고정 URL** 필요

## 기술 스택 변경

| 구분 | 기존 (앱) | 웹 이전 후 |
|---|---|---|
| 프레임워크 | React Native + Expo | **React 19 + Vite** |
| 언어 | TypeScript | TypeScript (동일) |
| 라우팅 | expo-router (파일 기반) | **react-router-dom v7** |
| UI 요소 | View / Text / Image | div / span / img |
| 스타일 | StyleSheet 객체 | **CSS 파일 + CSS 변수** |
| 아이콘 | @expo/vector-icons (Ionicons) | **react-icons/io5** + 인라인 SVG |
| 벡터 그래픽 | react-native-svg | 인라인 `<svg>` |
| 토큰 저장 | expo-secure-store | **localStorage** |
| 소셜 로그인 | expo-auth-session / web-browser | **window.location 리다이렉트** (웹 OAuth 콜백 `/auth`) |
| 이미지 선택 | expo-image-picker | **`<input type="file">`** |
| HTML 본문 렌더 | react-native-render-html | **dangerouslySetInnerHTML + CSS** |
| 모달 | RN Modal / AppModal | **포털 오버레이 컴포넌트** |
| 번들러 | Metro | **Vite (esbuild + rollup)** |
| 배포 | EAS (URL 매번 변경) | **GitHub + Vercel 자동배포 (고정 URL)** |
| 백엔드 | ilaw-backend | 동일 (변경 없음) |

> 로직·API 호출·상태관리 코드는 대부분 **그대로 재사용**, 바뀐 건 주로 화면(JSX/스타일)과 네이티브 API 치환.

## 화면 프레임 (중요)
원본 Expo 웹과 동일하게 **390×844 고정 캔버스를 `transform: scale`로 균일 축소**.
→ 어떤 폰·데스크탑에서든 **같은 비율·같은 디자인**으로 보임 (Figma 비율과 일치). 창 크기에 맞춰 JS로 스케일 계산.

## 폴더 구조 (airo-web)
```
src/
  api/        client, auth, qa, community, manual, home, notifications, ai, upload, types
  context/    AuthContext, NotificationSettingsContext
  components/ TabBar, Overlay(모달), HighlightText
  routes/     27개 화면 (.tsx + .css)
  data/       qnaData (변호사 Q&A 정적 데이터)
  index.css   전역 팔레트/폰트/프레임
```

## 인증 / 백엔드 연동
- **JWT Bearer**, 401 시 `/auth/refresh`로 자동 갱신 (fetch 래퍼)
- 웹 OAuth: `GET /auth/{provider}?redirectUri=https://.../auth` → 콜백이 토큰을 쿼리로 전달 → `localStorage` 저장
- 백엔드 CORS 전체 허용, 헤더 방식이라 웹에서 그대로 호출
- **소셜 로그인은 https 배포 주소에서만 동작** (백엔드가 http://localhost 리다이렉트 미허용)

## 폰트
- 본문: 모바일 앱과 동일하게 **Noto Sans KR** 웹폰트 강제 (OS 무관 동일 렌더)
- 스플래시/타이틀: 커스텀 **AiroFont**(font.ttf)만 부분 적용

## 이전한 화면 (27개, 전부 백엔드 연결)
스플래시 · 로그인 · OAuth콜백 · 온보딩 / 홈(검색·인기·AI) / 매뉴얼(카테고리·목록·상세HTML·도움받기) /
Q&A(목록·상세·질문·답변) / 커뮤니티(목록·상세·글쓰기) / 프로필·정보수정 /
내 질문·답변·스크랩·스크랩Q&A / 알림·알림설정 / AI챗 / 약관·개인정보

## 배포
- GitHub: `iLaw-app/iLaw-web` (public)
- Vercel 연결 → `git push` 하면 **`i-law-web.vercel.app`에 자동 배포** (URL 고정)
